## 计费模式总览

平台托管模式下支持两种付费方式，全局二选一：

1. **预充值按量计费** — 充值买积分，按实际 token 消耗扣积分
2. **周/月度订阅** — 固定价格，有效期内所有功能不限量使用

BYOK（自带 API Key）用户完全不受计费系统影响，无限免费使用。

---

## 积分体系

- 计费单位：**积分（credits）**
- 兑换比例：**1 元 = 100 积分**
- Token 定价：
  - **Input tokens**：3 积分 / 千tokens
  - **Output tokens**：12 积分 / 千tokens

## 计费流程

```
用户发起 API 请求
    │
    ├── BYOK 模式？ ──是──→ 直接放行
    │
    ├── 有效订阅？ ──是──→ 直接放行
    │
    ├── 刷题且有免费额度？ ──是──→ 放行，扣免费次数
    │
    ├── 余额 >= 预估消耗？ ──是──→ 放行
    │   │                          │
    │   │                      API 调用完成
    │   │                          │
    │   │                    取实际 token 数
    │   │                          │
    │   │                    计算积分并扣除
    │   │
    │   └── 否 ──→ 返回 403："余额不足，请充值或订阅"
```

| 字段 | 类型 | 说明 |

| 新增字段 | 类型 | 说明 |
|---------|------|------|
| credits_cost | int, nullable | 该次调用扣除的积分 |

### 后端核心逻辑

**计费守卫**（在 `deps.py` 中新增）：
- 每个需要 AI 调用的接口注入计费检查
- BYOK 直接放行；订阅中直接放行；免费刷题扣次数放行；否则检查余额

**扣费逻辑**：
- API 调用完成后，从 OpenAI response 的 `usage` 字段取实际 token 数
- 思考模型（如 Gemini 3 Pro Preview）的 `total_tokens` 包含不可见的思考 tokens，需按以下方式计算：
  - 实际输出消耗 = `total_tokens - prompt_tokens`（含可见响应 + 不可见思考）
  - 思考 tokens = `total_tokens - prompt_tokens - completion_tokens`
- 计算积分：`prompt_tokens / 1000 * 3 + (total_tokens - prompt_tokens) / 1000 * 12`
- 从 `user_wallets.balance` 中扣除
- 在 `usage_records` 中记录 `credits_cost`

### 后端新增文件/路由

**`backend/app/models/billing.py`** — ORM 模型：
- UserWallet、Subscription、RechargeRecord

**`backend/app/services/billing_service.py`** — 计费核心逻辑：
- `check_access(user_id, feature, db)` — 检查用户是否有权调用
- `deduct_credits(user_id, input_tokens, output_tokens, db)` — 调用后扣费
- `get_wallet(user_id, db)` — 获取钱包信息
- `recharge(user_id, tier, db)` — 充值
- `subscribe(user_id, plan_type, db)` — 订阅

**`backend/app/api/billing.py`** — 计费路由：
- `GET /api/billing/wallet` — 获取钱包余额、免费额度、订阅状态
- `POST /api/billing/recharge` — 充值（第一版模拟充值）
- `POST /api/billing/subscribe` — 订阅（第一版模拟订阅）
- `GET /api/billing/records` — 充值/消费记录

### 前端改动

**设置页（SettingsPage.jsx）**：
- 选择平台模式后弹出计费方案选择（预充值 / 订阅）
- 预充值：显示 3 个档位按钮，点击模拟充值
- 订阅：显示周卡/月卡按钮，点击模拟订阅

**钱包/账单展示**（嵌入设置页或独立区域）：
- 当前余额、订阅状态、免费刷题剩余
- 充值/消费记录列表

**功能页面拦截提示**：
- 平台模式 + 无余额 + 无订阅时，显示"余额不足，请充值或订阅"
