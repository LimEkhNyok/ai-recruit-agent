# 本地代码更新到服务器 — 操作手册

> 服务器地址：`13.214.147.27`（AWS EC2 Ubuntu）
> 密钥文件：`D:\BrowserDownload\database-5-key.pem`

## 先判断：这次改了什么？

每次更新前，先想清楚改了哪部分，不同部分的更新步骤不同：

| 改动位置 | 需要做什么 |
|----------|-----------|
| 只改了 `backend/` | 步骤 1 → 2 → 3 |
| 只改了 `frontend/` | 步骤 1 → 2 → 4 → 5 |
| 前后端都改了 | 步骤 1 → 2 → 3 → 4 → 5 |

---

## 步骤 1：本地提交并推送到 GitHub

在本地项目根目录打开终端（PowerShell 或 WSL 都可以）：

```bash
cd C:\Users\72409\Desktop\ai-recruit-agent

# 查看改了哪些文件
git status

# 把要提交的文件加入暂存区（把 文件路径 替换为实际文件）
git add 文件路径1 文件路径2

# 提交（替换为你的提交信息）
git commit -m "这里写你的提交说明"

# 推送到 GitHub
git push
```

> **注意**：不要用 `git add .` 或 `git add -A`，避免把 `.env`（含密码）等敏感文件意外提交。

## 步骤 2：服务器拉取最新代码

用终端 SSH 连接服务器：

```powershell
ssh -i D:\BrowserDownload\database-5-key.pem ubuntu@13.214.147.27
```

连上后执行：

```bash
cd ~/ai-recruit-agent
git pull
```

> 如果提示冲突，说明有人直接在服务器上改过代码。一般不要在服务器上直接改代码，所有改动都应该从本地提交。

## 步骤 3：重启后端（改了 backend/ 才需要）

```bash
sudo systemctl restart ai-recruit-backend
```

验证是否启动成功：

```bash
# 查看服务状态，应该显示 active (running)
sudo systemctl status ai-recruit-backend

# 测试接口是否正常
curl http://127.0.0.1:8000/api/health
# 应返回 {"code":0,"data":{"status":"ok"},"message":"ok"}
```

如果启动失败，查看错误日志：

```bash
sudo journalctl -u ai-recruit-backend --since "2 min ago" --no-pager
```

> **常见问题**：如果新增了 Python 依赖包（改了 `requirements.txt`），需要先安装：
> ```bash
> cd ~/ai-recruit-agent/backend
> source venv/bin/activate
> pip install -r requirements.txt
> sudo systemctl restart ai-recruit-backend
> ```

## 步骤 4：本地构建前端（改了 frontend/ 才需要）

**为什么不在服务器上构建？** 因为服务器内存只有 1GB，构建前端时会内存不足导致崩溃。所以我们在本地构建好，再把产物传上去。

在本地 **PowerShell** 中执行：

```powershell
cd C:\Users\72409\Desktop\ai-recruit-agent\frontend
npm run build
```

看到类似下面的输出就说明构建成功：

```
✓ 3748 modules transformed.
dist/index.html                     0.48 kB
dist/assets/index-xxxxx.css         9.92 kB
dist/assets/index-xxxxx.js       2397.96 kB
✓ built in 15.29s
```

构建产物在 `frontend/dist/` 目录下。

---

## 步骤 5：上传前端产物到服务器（改了 frontend/ 才需要）

在本地 **PowerShell** 中执行（注意 `dist\` 和 `ubuntu@` 之间有空格）：

```powershell
scp -i D:\BrowserDownload\database-5-key.pem -r C:\Users\72409\Desktop\ai-recruit-agent\frontend\dist\ ubuntu@13.214.147.27:~/ai-recruit-agent/frontend/
```

然后 SSH 到服务器，修复文件权限（否则 nginx 读不到文件会报 500 错误）：

```bash
chmod -R 755 /home/ubuntu/ai-recruit-agent/frontend/dist
```

## 踩坑记录

### 1. 上传后访问报 500 错误

**原因**：nginx 运行用户（`www-data`）没有权限读取 `dist/` 目录下的文件。

**解决**：每次 scp 上传后都要执行：
```bash
chmod -R 755 /home/ubuntu/ai-recruit-agent/frontend/dist
```

### 2. 后端启动失败：ModuleNotFoundError

**原因**：新增了 Python 依赖但没在服务器上安装。

**解决**：
```bash
cd ~/ai-recruit-agent/backend
source venv/bin/activate
pip install -r requirements.txt
sudo systemctl restart ai-recruit-backend
```

### 3. 后端报错：bcrypt 相关错误

**原因**：`passlib` 和新版 `bcrypt` 不兼容。

**解决**：
```bash
cd ~/ai-recruit-agent/backend
source venv/bin/activate
pip install bcrypt==4.0.1
sudo systemctl restart ai-recruit-backend
```

### 4. scp 命令报 usage 错误

**原因**：命令中路径和目标地址之间缺少空格，或密钥路径不对。

**正确格式**（注意 `dist\` 后面有空格再接 `ubuntu@`）：
```powershell
scp -i 密钥路径 -r 本地路径\ ubuntu@服务器地址:远程路径/
```

### 5. 浏览器看到的还是旧页面

**原因**：浏览器缓存了旧的 JS/CSS 文件。

**解决**：按 `Ctrl+Shift+R` 强制刷新，或者打开无痕窗口访问。

### 6. git pull 冲突

**原因**：有人在服务器上直接修改了文件。

**解决**：
```bash
# 丢弃服务器上的本地修改（确认这些修改不需要保留）
git checkout -- .
git pull
```

> **原则**：永远不要在服务器上直接改代码。所有修改都在本地完成后推送。
