# 产品改进计划

## 一、刷题练习：接入经典题库知识点 + AI 生成题目

### 思路

现有刷题模块完全由 AI 随机出题，题目质量和知识点覆盖不可控。改进方向是引入经典题库（如 LeetCode）的**知识点分类体系**，由 AI 根据知识点生成原创题目。

### 方案

1. 从 GitHub 开源题库（如 [azl397985856/leetcode](https://github.com/azl397985856/leetcode/tree/master/problems)）提取知识点标签体系（哈希表、双指针、动态规划、树、图等）
2. 建立知识点分类索引，包含：知识点名称、难度分级、关联考点、典型解题思路
3. 用户选择知识点后，AI 根据该知识点**原创生成题目**，而非搬运原题（规避版权风险）
4. 用户答错或跳过时，AI 基于同一知识点**举一反三**，生成相似但不同的变体题，强化薄弱环节
5. 知识点掌握程度可视化，帮助用户了解自己的薄弱区域

### 实现要点

- 只提取知识点分类和解题思路，不复制原题描述
- 题库提供**知识点索引**，AI 负责**生成具体题目**
- 与现有 AI 出题模式并存，用户可切换"知识点刷题"或"AI 随机出题"

## 二、模拟面试：支持语音输入

### 思路

当前模拟面试只支持文字输入，与真实面试体验差距大。接入浏览器原生语音识别 API（Web Speech API），让用户可以语音作答，接近真实面试场景。

### 方案

1. 面试页输入框旁增加麦克风按钮，点击开始语音识别
2. 使用浏览器原生 `Web Speech API`（免费、零成本、无需后端）
3. 语音识别结果实时填入文本框，用户可在提交前编辑修正
4. 支持中文识别（`lang: 'zh-CN'`）

### 核心代码参考

```js
const recognition = new webkitSpeechRecognition()
recognition.lang = 'zh-CN'
recognition.continuous = true

recognition.onresult = (event) => {
  const text = event.results[event.results.length - 1][0].transcript
  // 实时填入输入框
}

recognition.start()  // 开始录音
recognition.stop()   // 停止录音
```

### 注意事项

- Chrome/Edge 支持最好，Safari 部分支持，Firefox 不支持
- 中文识别准确率约 85-90%，日常对话够用，专业术语可能有偏差
- 依赖 Google 语音服务，国内网络可能不稳定，后续可考虑讯飞/阿里语音 API 作为备选
- 摄像头面试作为远期规划，需要表情/姿态分析才有实际价值，等语音面试跑通后再考虑

## 三、岗位匹配：粘贴 JD → 分析考点 → 针对性模拟面试

### 思路

现有岗位匹配基于用户画像推荐岗位，但缺少与真实招聘需求的联动。新增"粘贴目标岗位JD"功能，让用户粘贴目标岗位的招聘描述，AI 分析考点后直接进入针对性模拟面试，形成完整闭环。

### 方案

1. 岗位匹配页新增"粘贴目标岗位JD"文本区域
2. 用户粘贴岗位 JD 后，调用 AI 分析提取：
   - 技术栈要求（如 React、Java、MySQL）
   - 核心能力要求（如系统设计、团队协作）
   - 加分项（如开源经验、大厂背景）
3. 以列表形式展示分析结果
4. 提供"针对此岗位模拟面试"入口，AI 根据提取的考点生成面试题
5. 复用现有面试模块，考点来源从通用切换为 JD 定向

### 用户流程

```
粘贴 JD → AI 提取考点和要求 → 展示分析结果 → 点击"针对性模拟面试" → 定向出题
```

---

## 建议实施顺序

| 优先级 | 功能 | 理由 |
|--------|------|------|
| 1 | 粘贴 JD → 针对性面试 | 改动最小，复用现有面试模块，串联价值最大 |
| 2 | 语音输入 | 前端改动为主，浏览器 API 零成本 |
| 3 | 知识点题库 + AI 出题 | 需要知识点体系整理 + 数据库存储，工作量最大 |
