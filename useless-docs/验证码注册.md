# 注册功能改造：邮箱验证码 + 密码

## 背景和要求

### 背景

当前系统注册流程为：用户填写姓名、邮箱、密码即可直接注册，无任何邮箱真实性验证。需要改造为**邮箱验证码注册**模式，确保用户邮箱真实有效。登录流程保持邮箱 + 密码不变。

### 技术栈现状

| 层级 | 技术 |
|------|------|
| 后端 | FastAPI + SQLAlchemy(async) + MySQL + Redis |
| 前端 | React 19 + Vite + Ant Design 5 + Zustand |
| 认证 | JWT access_token(7天) + refresh_token(30天) + bcrypt |

### SMTP 配置

- 服务商：阿里云邮件推送
- SMTP 地址：`smtpdm-ap-southeast-1.aliyun.com`
- 端口：`465`（SSL 加密）
- 发信地址 / 用户名：`noreply@smartmeal.longstock.com`
- 密码：通过环境变量 `SMTP_PASSWORD` 配置

### 改造要点

1. **注册流程**：邮箱 → 发送验证码 → 用户填写验证码 + 姓名 + 密码 → 完成注册
2. **登录流程**：保持现有邮箱 + 密码方式不变
3. **验证码存储**：使用 Redis，key 格式 `verify_code:{email}`，有效期 5 分钟
4. **发送频率限制**：同一邮箱 60 秒内只能发送一次验证码
5. **验证码格式**：6 位纯数字
6. **安全要求**：验证成功后立即删除 Redis 中的验证码，防止重放

### 涉及文件

**后端需修改/新增：**
- `backend/.env` / `backend/.env.example` — 添加 SMTP 环境变量
- `backend/requirements.txt` — 添加 `aiosmtplib` 依赖
- `backend/app/config.py` — 添加 SMTP 和验证码相关配置
- `backend/app/services/email.py` — **新增** 邮件发送服务
- `backend/app/services/verify_code.py` — **新增** 验证码生成、存储、校验服务
- `backend/app/schemas/auth.py` — 添加发送验证码和验证码注册的请求模型
- `backend/app/api/auth.py` — 添加发送验证码端点，修改注册端点

**前端需修改：**
- `frontend/src/api/auth.js` — 添加发送验证码 API
- `frontend/src/store/useAuthStore.js` — 更新注册方法签名
- `frontend/src/pages/RegisterPage.jsx` — 改造注册表单（分步/验证码输入）
